<p class="lede">Everything Signposts needs lives in one file at your repo root, organised around the thing you actually adopt and share — the <strong>bundle</strong>. Each bundle is one contiguous, cherry-pickable block.</p>

| Section | Holds | Read by |
| --- | --- | --- |
| `project` | Stack identity — a name and description. | the CLI |
| `config` | Engine runtime settings — currently `drift_tokens` (the sign re-injection cadence). | the sign injector |
| `bundles` | The unit of value. Each key is a bundle (its namespace); each block carries up to five keys. | everything |

## A bundle is one block

The old layout scattered a namespace across five top-level sections (`signs:`, `rules:`, `settings:`, `packs:`, `install:`) — cherry-picking one meant harvesting from all five. Bundle-first puts each bundle in **one place**. Copying a bundle between repos is copying its block plus its `rules/<bundle>/` folder — nothing else. The five keys a bundle may carry:

| Key | What it is |
| --- | --- |
| `from` | Provenance pin — **vendored bundles only** (`github:owner/repo#ref` or a local path). The base `refresh` diffs against. Local bundles simply omit it. |
| `signs` | Steering <Chip kind="sign">sign</Chip> notes. `at: session \| touch \| turn` (default `touch`). A `session` sign is [orientation](/docs/signs). |
| `rules` | <Chip kind="rule">rule</Chip> blocks. `use:` names a script; config rides inline. `at: write \| commit \| delete \| turn` (default `[write, commit]`). |
| `settings` | Host-permission entries merged into `.claude/settings.json` on install (`deny` auto-applied; `allow` surfaced for you). |
| `needs` | *(rare, on a rule)* an npm dep that exists only because that rule does. Baseline tools ride the engine, not bundles. |

A bundle **owns** `rules/<bundle>/**` — that ownership convention is how install and uninstall know what to copy and remove, so there's no machine manifest to drift. Gone from the old schema: `packs:`, `install: files`, install dates.

## A worked example

```yaml
project:
  name: my-core
  description: My personal dev guardrails. Reused on every repo; irrelevant rules sleep.

config:
  drift_tokens: 200000          # re-inject a sign after ~this many tokens of drift

bundles:

  # ── a LOCAL bundle — born here, may propagate upstream one day ──
  core:
    signs:
      - id: orientation
        at: session                        # the composed-CLAUDE.md slot (session start)
        text: |-
          docs/arch/ is the system prior — read before structural work.
          Repeated commands live in justfile recipes, never package.json scripts.
      - id: db-area
        globs: ["src/db/**"]               # at: touch is the default for signs
        text: "Queries go through the repository layer, never inline in a route."
    rules:
      - id: no-edit-generated
        use: core/protected-path
        deny: ["**/*.generated.ts", "vendor/**"]   # at: [write, commit] is the default
        message: "Generated / vendored — change the source, then regenerate."

  # ── a VENDORED bundle — pinned provenance, content editable in place ──
  fcis:
    from: github:you/hub#v1.4.2
    rules:
      - id: import-fence
        use: core/depcruise                # the layers-&-fences dialect (see below)
        layers:
          core:    ["src/lib/**/domain.ts"]
          effects: ["src/lib/**/db.ts"]
        forbid:
          - { from: core, to: effects, transitive: true, why: "purity is transitive" }
    settings:
      permissions:
        deny: ["Read(.env.keys)"]          # merged into .claude/settings.json on install
```

## How a rule reads

Each entry names a **script** with `use:` and carries that script's config inline. The whole entry is handed to the script **verbatim**.

- `use: core/<name>` — a [core script](/docs/rules); or `<bundle>/<name>` for [your own](/docs/authoring).
- `on:` — the glob(s) it applies to (some scripts use `deny:` or `run:` instead).
- `at:` — the [moment](/docs/wiring) it fires on. Rules default to `[write, commit]`; reach for `[commit]` (a whole-tree tool-gate), `[delete]`, or `[turn]` (end of turn). The legacy `when:` still loads.
- `message:` — the reason, fed back to the agent on a block.

<Note>
**Code patterns are files, not inline.** ast-grep rules stay as `rules/ast-grep/*.yml` (so the engine and `signposts test` use the same files); the `core/ast-grep` runner executes them in-process. Import fences ride the declarative [`core/depcruise`](/docs/rules/depcruise) dialect. Everything else is a normal `rules:` entry.
</Note>

## Backwards compatibility

A load-time normaliser accepts **both** shapes — bundle-first and the old section-first (`signs:` / `rules:` / `settings:` grouped by namespace, plus legacy `advisory:` and flat lists) — and hands the engine the flat form it already consumes. The **filename** is equally forgiving: Signposts prefers `signposts.yml` (what a fresh scaffold writes) but still reads a legacy `signposts.yaml`. Old files keep working forever; a repo moves to bundle-first `.yml` when you touch it. See [Packs & distribution](/docs/packs) for the install / refresh / uninstall flow.
